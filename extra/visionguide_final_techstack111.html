8 Layers 100% Free Stack Quant × AI Theme Runs Fully Local
Layer 1 Voice Input ▶ FREE · Local
OpenAI WhisperPyAudio

User speaks a goal ("Help me reach the door"). PyAudio captures mic stream in real time. Whisper (small model, ~150MB) runs locally — no internet, no latency, no cost — and transcribes to text. Output is a clean string fed to the LLM agent.

Input:Raw audio bytes from microphone
Output:Transcribed text string, e.g. "Help me reach the door"
Why local:Faster, no API key, works offline
Layer 2 LLM Agent Brain ▶ FREE Tier · Groq
Groq APILLaMA 3.1 8Bpython-dotenv

The core "agentic" component judges are looking for. Receives the user's goal + current scene context (detected objects + distances). Breaks the goal into subtasks, decides which objects to look for next, and generates spoken guidance. Runs in a loop — no re-prompting needed.

Input:User goal text + scene description (objects, distances from quant engine)
Output:JSON: { "speak": "...", "detect_next": "door handle", "action": "move_forward" }
Why Groq:Fastest free LLM API — ~300 tokens/sec. Free tier: 30 req/min.
Agent loop:Speak → detect → measure → re-plan → speak again, until goal reached
Layer 3 Camera Feed ▶ FREE · Local
OpenCVNumPy

Captures webcam frames at 10–15 FPS. Each frame is a NumPy array (H×W×3 BGR). OpenCV handles resizing to standard inputs for YOLO-World (640×640) and MiDaS (384×384). Same frame feeds both detection and depth simultaneously — parallel, not sequential.

Input:Webcam device (USB or laptop cam)
Output:NumPy array frame fed to Layer 4 and Layer 5 in parallel
Frame rate:10–15 FPS (sufficient for navigation guidance)
Layer 4 Open-Vocab Detection ▶ FREE · Local
YOLO-WorldUltralyticsPyTorch

Directly satisfies PS1: "new objects can be added without training". YOLO-World takes any text label at runtime and detects matching objects — zero retraining, zero fine-tuning. When LLM says "detect_next: door handle", YOLO-World finds it.

Input:Camera frame + list of text labels from LLM (e.g. ["door", "chair", "person"])
Output:Bounding boxes [x1,y1,x2,y2] + label + confidence score per object
Key feature:Open-vocabulary = no retraining. Detects ANY object by name on the fly.
Model size:~50MB (YOLO-World-S, fast variant)
Layer 5 Depth Estimation ▶ FREE · Local
MiDaSPyTorch HubNumPy

Satisfies PS1 requirement for "Depth estimation (Monocular)". MiDaS takes a single RGB frame and outputs a depth map — brighter = closer. Combined with YOLO-World bounding boxes, computes approximate real-world distance to any detected object.

Input:RGB camera frame (384×384)
Output:Depth map array (same H×W), relative depth values 0–1
Distance calc:Mean depth value within object bounding box → scaled to metres
Quant link:Feeds into Kalman Filter and Euclidean distance in Layer 6
Layer 6 ★ Quantitative Engine ▶ JUDGES LOVE THIS
NumPySciPyheapq (stdlib)

The component that locks in the Quant × AI theme. Four distinct mathematical algorithms, each with a clear quantitative purpose — judges cannot miss this layer.

① Kalman Filter:Tracks each detected object across frames. Estimates position even when temporarily occluded. Uses SciPy's linear algebra.
② A* Pathfinding:Builds a 2D grid from the depth map. Marks obstacle cells (close objects) as blocked. Finds shortest safe path to the target using heapq (Python stdlib).
③ Euclidean Dist:Computes real distance from camera to target using mean depth region. sqrt((Δx)² + (Δy)² + (Δz)²) on the 3D point cloud.
④ Conf. Threshold:Statistical filtering — only YOLO detections with confidence ≥ 0.6 pass to the agent. Eliminates noise.
Layer 7 Voice Output (TTS) ▶ FREE · Offline
pyttsx3

Converts LLM agent's text guidance into spoken audio. Runs fully offline using system TTS engine (SAPI5 on Windows, NSSpeechSynthesizer on macOS, espeak on Linux). No API calls, no latency beyond synthesis time (~0.5s).

Input:"Door detected 1.8 metres ahead, slightly to your left"
Output:Audio playback through speakers or headphones
Config:Rate: 150 wpm, Volume: 1.0, Voice: default system voice
Layer 8 Web Interface (HUD) ▶ FREE · Flask
FlaskSocket.IOHTML/CSS/JS

Anime-styled visual HUD for caregivers or sighted helpers. Shows live camera feed with YOLO bounding boxes, depth heat map, Kalman tracking trails, A* path overlay, and LLM agent's current spoken text. Flask serves the page; Socket.IO streams detection data to the browser in real time.

Why web:No install for judges — open browser → localhost:5000 → full demo.
Anime UI:Dark HUD, glowing bounding boxes, kanji accent (視覚), scanline overlay, neon trails for Kalman paths
Judging tip:Judges SEE the quant working — depth map, path overlay, tracking trails all visible live
